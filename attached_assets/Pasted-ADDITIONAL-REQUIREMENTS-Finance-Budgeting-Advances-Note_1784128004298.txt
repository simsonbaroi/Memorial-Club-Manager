IMPORTANT DEVELOPMENT RULES
Never delete financial records.
Always use audit logs.
Use soft delete only where appropriate.
Every financial change must be traceable.
Every report must be reproducible.
Keep all modules loosely coupled and modular.
Optimize for long-term maintainability.
Ensure all functionality works offline first and synchronizes safely when connectivity is available.
Maintain a clean, responsive, and professional user interface suitable for long-term use by a non-profit organization.
